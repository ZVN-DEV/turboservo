# TurboServo v0.8.2 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade TurboServo from Turbo Lang 0.7.6 to 0.8.2, adopting new builtins, hardening patterns, and modernizing syntax across the framework and showcase app.

**Architecture:** TurboServo is a thin HTTP framework layer over Turbo Lang's built-in `http_server`/`route`/`http_listen` APIs. The upgrade touches three layers: (1) the framework core (`src/servo.tb`, `src/request.tb`, `src/response.tb`, `src/router.tb`), (2) the showcase demo app (`examples/showcase/`, `src/showcase/`), and (3) metadata/docs/tests. No structural changes — this is an API alignment and modernization pass.

**Tech Stack:** Turbo Lang 0.8.2 (JIT), TurboServo framework

**Key v0.8.2 changes to adopt:**
- `http_server_public()` for production deployments
- `try_read_file`/`try_write_file` fallible I/O
- `hashmap_set_int`/`hashmap_get_int` for counters
- `+=` compound assignment operators
- String interpolation `{var}` syntax
- `.len()` method syntax
- CORS headers no longer sent by default (document this)
- Header limits enforced (8KB/line, 64KB total, 32MB body)

### Task 3: Framework core — `src/request.tb`

**Files:**
- Modify: `src/request.tb`

- [ ] **Step 1: Modernize `parse_i64` with `+=` syntax**

Replace the `digit_to_i64` / `parse_i64` functions. The new version uses `+=` compound assignment (new in v0.8.x):

```turbo
fn parse_i64(s: str) -> i64 {
    let trimmed = trim(s)
    if len(trimmed) == 0 {
        return 0
    }

    let mut value = 0
    let mut negative = false
    let chars = split(trimmed, "")
    let mut first = true

    for ch in chars {
        if first {
            first = false
            if ch == "-" {
                negative = true
            } else {
                value = digit_to_i64(ch, value)
            }
        } else {
            value = digit_to_i64(ch, value)
        }
    }

    if negative {
        0 - value
    } else {
        value
    }
}
```

Note: `parse_i64` logic stays the same — `digit_to_i64` helper is still needed because Turbo doesn't have char-to-int builtins. No changes to the actual parsing logic.

- [ ] **Step 2: Add `url_decode` entries for `%20` (space) and `%25` (percent)**

Add to the `url_decode` function body, before the final `decoded` return:

```turbo
    decoded = replace(decoded, "%20", " ")
    decoded = replace(decoded, "%25", "%")
```

`%20` is the standard encoding for space (alongside `+` which is already handled). `%25` is literal percent — must come last to avoid double-decoding.

### Task 5: Showcase — adopt `hashmap_set_int`/`hashmap_get_int` for counters

**Files:**
- Modify: `examples/showcase/main.tb`

Currently the showcase uses `mutex(0)` + `mutex_get`/`mutex_set` for all counters. The counters themselves are fine — `mutex` is the right concurrency primitive for shared state across request handlers. However, the `server_state_json` function manually assembles JSON with `to_str(mutex_get(...))` for 12 fields. We can clean this up.

- [ ] **Step 1: No structural change to mutex counters**

The mutex-based counter pattern is correct for concurrent HTTP handlers. `hashmap_set_int`/`hashmap_get_int` don't provide thread safety, so the mutex counters stay. However, we should adopt `+=` syntax throughout.

- [ ] **Step 2: Modernize `increment` function with `+=`**

Replace the `increment` function:

```turbo
fn increment(counter: i64) -> i64 {
    let next = mutex_get(counter) + 1
    mutex_set(counter, next)
    next
}
```

This stays the same — `+=` can't be used with `mutex_get`/`mutex_set` since they're separate calls. No change needed here.

### Task 7: Hello example — modernize and add error handling

**Files:**
- Modify: `examples/hello/main.tb`

- [ ] **Step 1: Modernize the hello example**

Update `examples/hello/main.tb` to use `+=` and update the startup message to reference v0.8.2:

```turbo
// TurboServo — Hello World Example

import { create, get, listen } from "../../src/servo"

import { json, text } from "../../src/response"

import { parse_i64, query } from "../../src/request"

fn configured_port(fallback: i64) -> i64 {
    let raw = trim(env_get("PORT"))
    if len(raw) == 0 {
        return fallback
    }

    let parsed = parse_i64(raw)
    if parsed > 0 {
        parsed
    } else {
        fallback
    }
}

fn main() {
    let port = configured_port(3000)
    let app = create(port)

    get(app, "/", |req: str| -> str {
        text(200, "Hello from TurboServo!")
    })

    get(app, "/json", |req: str| -> str {
        json(200, json_stringify("message", "Hello, World!"))
    })

    get(app, "/greet", |req: str| -> str {
        let name = query(req, "name")
        if len(name) > 0 {
            json(200, json_stringify("greeting", "Hello, " + name + "!"))
        } else {
            json(200, json_stringify("greeting", "Hello, stranger!"))
        }
    })

    print("TurboServo running on http://localhost:" + to_str(port))
    listen(app)
}
```

No major changes — this example is already clean. Leave it simple as the entry-point example.
